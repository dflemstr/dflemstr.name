How long did it take for this page to load? Did you even notice it? What actually happened when you clicked that link?

Even though this is a super simple web site, I've tried to make it as advanced as possible, hopefully demonstrating some good engineering practices while we're at it.

Of course, this is all super overkill for a site of this size; any reasonable person would have just hand-edited some HTML and put it on some shared hosting. However, that won't do for large-scale deployments, which is what I deal with on a daily basis.

Imagine for a second that this is a large-scale web site serving thousands of users concurrently. That's what this web site was designed to be able to handle.

Oh, and I've tried to keep the buzzword count high! (No, don't take this page too seriously)

A series of tubes or whatever they call it these days

You're connected to a machine hosted by DigitalOcean in Amsterdam, datacenter 3.

The server that served you this web page is part of a CoreOS cluster running Fleet. There's probably only one server in that cluster right now (Namely core-0.dflemstr.name), because I like money, but it could scale to any number of machines.

This cluster is running an unit defining a Docker container to run, based off of this image.

Commit to Deploy

Of course, the source code for the web site is hosted in a Git repository available here.

On every commit, the web site is automatically built, tested and deployed using Docker Automated Builds. Additionally, tests are run separately by TravisCI, and you can see the results here.

Modern web development

The source code for the web page is managed using Yeoman, build dependencies are fetched using Grunt and the actual web site uses components managed by Bower.

Of course, the website manages cache correctly, as well as minification and compilation of resources.

Responsive

The page is being fetched in parts using AngularJS, and the view for this article is rather tiny. That's the main secret behind why it loads so fast!

Of course, this is also the case on most available platforms where you'll find a web browser. The page scales to any screen size and resources are fetched in a way that optimizes bandwidth and delivers the correct resolutions for your device.
